<Callout>
  **Note:** The installation process follows the same structure as
  [shadcn/ui](https://ui.shadcn.com/docs/installation).
</Callout>

<Steps>
<Step>

### Initialize shadcn/ui

Run the `init` command to create a new project with shadcn/ui or set up an existing one.

<CodeTabs
  codes={{
    pnpm: `pnpm dlx shadcn@latest init`,
    npm: `npx shadcn@latest init`,
    yarn: `yarn shadcn@latest init`,
    bun: `bunx --bun shadcn@latest init`,
  }}
/>

</Step>

<Step>

### Add the Azemmur registry

Open your project's `components.json` file and add the Azemmur registry to the `registries` field.  
This allows you to pull components directly from the Azemmur distribution.

```json
{
  "registries": {
    "@azemmur": "http://localhost:3000/r/{name}.json"
  }
}
```

</Step>

<Step>

### Add Components

Run the `add` command to add a component from the Azemmur registry.

<CodeTabs
  codes={{
    pnpm: `pnpm dlx shadcn@latest add @azemmur/components-button`,
    npm: `npx shadcn@latest add @azemmur/components-button`,
    yarn: `yarn dlx shadcn@latest add @azemmur/components-button`,
    bun: `bunx --bun shadcn@latest add @azemmur/components-button`,
  }}
/>

</Step>

<Step>

### Import Components

Import the component into your project.

```tsx
'use client';

import * as React from 'react';
import { cva, type VariantProps } from 'class-variance-authority';

import {
  Button as ButtonPrimitive,
  type ButtonProps as ButtonPrimitiveProps,
} from '@/registry/components/primitives/buttons/button';
import { cn } from '@workspace/ui/lib/utils';

const buttonVariants = cva(
  "inline-flex items-center justify-center gap-2 whitespace-nowrap rounded-md text-sm font-medium transition-[box-shadow,_color,_background-color,_border-color,_outline-color,_text-decoration-color,_fill,_stroke] disabled:pointer-events-none disabled:opacity-50 [&_svg]:pointer-events-none [&_svg:not([class*='size-'])]:size-4 shrink-0 [&_svg]:shrink-0 outline-none focus-visible:border-ring focus-visible:ring-ring/50 focus-visible:ring-[3px] aria-invalid:ring-destructive/20 dark:aria-invalid:ring-destructive/40 aria-invalid:border-destructive",
  {
    variants: {
      variant: {
        default:
          'bg-primary text-primary-foreground shadow-xs hover:bg-primary/90',
        accent: 'bg-accent text-accent-foreground shadow-xs hover:bg-accent/90',
        destructive:
          'bg-destructive text-white shadow-xs hover:bg-destructive/90 focus-visible:ring-destructive/20 dark:focus-visible:ring-destructive/40 dark:bg-destructive/60',
        outline:
          'border bg-background shadow-xs hover:bg-accent hover:text-accent-foreground dark:bg-input/30 dark:border-input dark:hover:bg-input/50',
        secondary:
          'bg-secondary text-secondary-foreground shadow-xs hover:bg-secondary/80',
        ghost:
          'hover:bg-accent hover:text-accent-foreground dark:hover:bg-accent/50',
        link: 'text-primary underline-offset-4 hover:underline',
      },
      size: {
        default: 'h-9 px-4 py-2 has-[>svg]:px-3',
        sm: 'h-8 rounded-md gap-1.5 px-3 has-[>svg]:px-2.5',
        lg: 'h-10 rounded-md px-6 has-[>svg]:px-4',
        icon: 'size-9',
        'icon-sm': 'size-8 rounded-md',
        'icon-lg': 'size-10 rounded-md',
      },
    },
    defaultVariants: {
      variant: 'default',
      size: 'default',
    },
  },
);

type ButtonProps = ButtonPrimitiveProps & VariantProps<typeof buttonVariants>;

function Button({ className, variant, size, ...props }: ButtonProps) {
  return (
    <ButtonPrimitive
      className={cn(buttonVariants({ variant, size, className }))}
      {...props}
    />
  );
}

export { Button, buttonVariants, type ButtonProps };
```

</Step>

<Step>

### Customize and Extend

> Make it yours — tweak, extend, and build on top of the foundation provided.

You’re now free to **adapt, customize, and style** your component however you like.  
The Azemmur registry gives you full access to the source — feel free to modify variants, add new props, or integrate it with your design system.

</Step>
